# Nebula

Nebula 是一款面向 macOS 的数据库与消息系统工作台。它把常见的数据浏览、结构管理、SQL 控制台、消息订阅、结果导出和 AI 辅助操作放在一个统一的桌面应用里，目标是让日常排查、查看、编辑和迁移数据更快、更安全。

> 说明：这个仓库仅作为 Nebula 安装包、在线更新和使用说明发布仓库使用，不包含应用源代码。

## 适合谁

- 需要同时管理多个 MySQL / TiDB / PostgreSQL 连接的后端、数据、测试和运维同学。
- 需要快速查看 MongoDB、Elasticsearch、Redis、Etcd、TDengine 等数据源的开发者。
- 经常查看 Kafka / MQTT 消息、调试 topic、解析 Proto 消息的同学。
- 希望用 AI 帮忙生成查询、解释数据、导出文件、分析结构变更，但又不希望 AI 直接绕过确认执行危险操作的用户。

## 下载安装

请在 [Releases](https://github.com/loveyu233/nebula/releases) 下载最新的 `Nebula-*.dmg` 安装包。

安装方式：

1. 下载最新 DMG。
2. 打开 DMG。
3. 将 `nebula.app` 拖入 `Applications`。
4. 第一次打开后，在设置中根据需要配置外观、存储、AI 和更新检查。

如果 macOS 提示安全确认，请确认安装包来自本仓库 Releases 页面后再打开。

| PostgreSQL | 数据浏览、schema 支持、表结构、执行角色、结构对比、SQL 控制台、日志、导出、AI 辅助 |
| MongoDB | 集合浏览、文档查看、查询与结果展示 |

## 快速开始

### 1. 创建连接

点击首页左侧或顶部的“新建连接”，选择连接类型并填写地址、端口、账号、密码等信息。

创建连接时的小技巧：

- 点击连接类型卡片的任意位置即可切换类型，不需要只点文字。
- 点击“测试连接”后，成功时会显示连接成功和版本信息。
- 连接可以分组；侧边栏会按分组展示，适合区分本地、测试、正式环境。

### 2. 选择数据库、schema 和表

进入连接后，在左侧资源区选择数据库和表。

PostgreSQL 会额外显示 schema 选择。选择 schema 后再展示表列表，避免误默认进入 `public`。

资源区支持右键快速复制：

- 右键数据库名称：复制数据库名。
- 右键 PostgreSQL schema 名称：复制 schema 名。
- 右键表名称：复制表名。
- 顶部当前位置里的表名和描述也支持右键复制，适合快速粘贴到 SQL、文档或沟通消息中。

Nebula 会记住你的常用操作状态，例如：

- 上次选择的数据库、schema、表。
- 数据浏览里的 `WHERE` 条件。
- 字段排序状态。
- 选择展示的列。
- 当前连接打开的功能页。

切换连接或重启 App 后，会尽量恢复到你上次的工作位置。

## 数据浏览

数据浏览是 Nebula 最常用的页面，用于查看、筛选、编辑和导出表数据。

### 表格操作

| 操作 | 效果 |
| --- | --- |
| 单击行 / 单元格 | 选中当前行或单元格 |
| 双击单元格 | 打开右侧编辑面板，查看和编辑完整字段内容 |
| 点击表头 | 按该列排序：升序 → 降序 → 取消排序 |
| 依次点击多个表头 | 组合多列排序，表头会显示排序顺序 |
| 悬浮表头 | 显示字段名、字段描述和字段类型 |
| 右键表头 | 复制当前列名到剪切板 |
| 右键单元格 | 将该单元格自动追加到 `WHERE` 条件并立即刷新 |
| `Option` + 右键单元格 | 使用 `OR` 追加筛选条件；默认右键使用 `AND` |

右键快速筛选会自动处理值类型：

- 字符串：生成类似 ``name = 'zs'``。
- 数字：生成类似 ``id = 10``，不会多加引号。
- 布尔值：按当前数据库类型生成合适的布尔字面量。
- `NULL`：生成类似 ``deleted_at IS NULL``。
- 已有条件时：默认追加为 `(<原条件>) AND <新条件>`。
- 按住 `Option` 时：追加为 `(<原条件>) OR <新条件>`。

### 列宽和表头显示

数据浏览表格支持更接近电子表格的列宽体验：

- 默认列宽会至少完整显示列名称，避免表头一打开就被截断。
- 拖动表头右侧边线可以调整当前列宽。
- 双击表头右侧边线会自动适配当前已加载数据的内容宽度。
- 每张表的列宽会单独持久化，切换表或重启后会恢复。
- 鼠标离开列边线后会自动退出列宽拖拽光标状态，避免误以为仍在调整列宽。

可以在设置里调整：

- 表头列名称的对齐方式：左侧、居中、右侧。
- 字段描述的对齐方式：左侧、居中、右侧。
- 不同字段类型在浅色 / 深色模式下的展示颜色，例如整数、浮点数、金额、字符串、JSON 等。

### WHERE 胶囊

数据浏览顶部的 `WHERE` 胶囊是筛选入口：

- 无条件时是普通状态。
- 有条件时会变成醒目的状态，表示当前数据正在被筛选。
- 点击整个 `WHERE` 胶囊可以清空条件并重新加载数据。
- 输入框获得焦点时会显示边框高亮，方便确认当前正在编辑筛选条件。

### 编辑数据

双击单元格后会从右侧打开编辑面板：

- 可查看字段名称、类型、描述、当前值和原始值。
- 长文本、JSON 等内容会更适合在侧栏中编辑。
- 提交修改前会显示待提交状态。
- 提交结果会在系统 toolbar 中提示，不会挤占表格区域。

### 导出选中数据

选中行后可以点击“导出选中数据”。支持：

- 导出为 SQL 文件。
- 导出为 Excel。
- 导出为 CSV。
- 导出为 TSV。
- 导出到同类型连接的目标数据库 / schema / 表。

导入到目标表前会校验表结构，避免把数据写入不一致的目标表。写入目标数据库时会尽量使用事务保证一致性。

## 表结构

表结构页用于查看和维护字段、索引、约束以及即将执行的 DDL。

常用能力：

- 查看字段、类型、默认值、是否可空、注释等信息。
- 添加新表。
- 复制当前表结构到剪切板。
- 添加字段、索引和约束。
- PostgreSQL 支持执行角色，可用于需要 `SET ROLE` 的 DDL 场景。
- 有未提交的 AI 或手工结构草稿时，左侧表列表会显示黄色提示点。
- 点击“提交修改”后执行结构变更。
- 在“将执行 SQL”区域可以直接“提交修改”或“放弃修改”。

新版表结构页更聚焦字段表格本身：

- 顶部只保留 `字段 / 索引 / 约束 / 待变更` 状态，减少无关信息。
- PostgreSQL 的执行角色放在同一行右侧，只保留必要输入框。
- 字段表格会占满页面可用高度，内容超出时在表格内部滚动。
- “将执行 SQL”默认可以收起，展开后查看真实 DDL。
- `放弃修改` 为红色操作，只有存在未提交变更时才可点击。
- 字段单元格不再显示多余的小图标，双击字段后进入右侧编辑侧栏。

安全原则：结构变更不会静默执行，涉及 DDL 的草稿需要用户在界面确认后再提交。

## SQL 控制台

控制台用于执行 SQL、查看结果和导出结果。

常用操作：

- 编写 SQL 后执行查询或变更。
- 控制台内容会持久化，切换页面再回来不会丢失正在写的 SQL。
- 执行结果表格支持单元格选择。
- 结果单元格支持右键复制。
- 结果卡片中的“导出结果”可复用数据浏览的导出能力，支持 Excel、CSV、TSV、SQL 等格式。

AI 生成的非查询 SQL 只会跳转到控制台形成草稿，不会替你点击执行。

## Kafka

Kafka 页面用于查看 Topic 和捕获消息。

使用细节：

- 点击“捕获”会重新拉取最新消息，而不是使用旧缓存。
- Topic 卡片支持置顶，多个置顶 topic 会按置顶时间倒序排列。
- 置顶状态会持久化，重启后仍保留。
- 支持导入 Proto 文件进行消息解码。
- 大消息或复杂解码会进入解码状态，避免界面被一次性大内容卡住。

## MQTT

MQTT 页面支持持续订阅 topic。

使用细节：

- 点击订阅后会持续接收消息。
- 切换连接或页面不会自动停止订阅。
- 只有手动取消订阅才会停止。
- toolbar 右侧会显示正在订阅的 topic 数量。
- 点击订阅提示胶囊可以快速跳回对应连接和 topic 页面。

## AI Command Center

Nebula AI 是一个面向数据库工作流的 Agent 入口，可以帮你查询数据、解释结构、生成 SQL、导出文件或规划操作步骤。

入口：

- 点击侧边栏或 toolbar 中的 Nebula AI。
- 快捷键：`Shift` + `Command` + `I`。

输入框操作：

| 操作 | 效果 |
| --- | --- |
| `Command` + `Return` | 发送当前问题 |
| `Esc` | 中断当前回复 |
| 回复中继续输入并发送 | 新问题会抢占旧问题 |
| 输入 `@` | 弹出当前数据库已加载表列表，可选择表结构作为本次上下文 |
| 添加文件 | 让 AI 分析文件、解析连接配置、生成导入 SQL 或生成导出文件 |
| 上 / 下方向键 | 在输入历史中切换上一条 / 下一条问题 |

AI 的安全边界：

- 查询类操作可以直接返回结果。
- 任何非查询操作都需要用户确认。
- 删除、更新、DDL、导入等危险操作不会静默执行。
- AI 生成的控制台 SQL 只是草稿，需要用户自己检查并执行。
- AI 生成文件后会提供“打开文件”和“打开所在目录”。

AI 回复中的表格：

- 会以本地表格组件展示。
- 超宽内容支持横向滚动。
- 超长内容支持纵向滚动或省略展示。
- 表格单元格可右键复制完整内容。
- 表格支持“导出表格”，可复用通用导出组件。

## 设置

设置页包含常用偏好：

- 外观：跟随系统、浅色、深色。
- 存储：配置数据库存储。
- AI：配置 AI 服务、模型、模式，并可检测配置是否可用。
- 更新：可开启“启动时自动检查更新”。开启后，每次打开 App 会静默检查新版本；没有新版本时不打扰，有新版本时再提示。

## 在线更新

Nebula 内置 Sparkle 在线更新。你可以在菜单中点击：

```text
nebula > 检查更新...
```

如果有新版本，应用会展示更新说明，并引导下载、安装和重启。

也可以在设置里开启“启动时自动检查更新”。开启后，App 每次启动会自动检查一次；没有新版本时不会弹窗打扰。

## 安全与确认

Nebula 的设计原则是：提高效率，但不替用户绕过关键确认。

- 数据删除、更新、DDL、导入等非查询操作需要确认。
- AI 不会直接替你执行危险 SQL。
- 导出到目标数据库前会进行结构校验。
- 数据库错误和后端请求错误会集中展示在系统 toolbar，避免页面上到处出现重复错误。
